# 课程文本规范与预处理流程

## 1. 文档目的

本文档冻结 `pdf_ingest` 面向 GraphRAG 的课程文本规范，作为后续预处理、导出、回归测试和验收的统一依据。

规范目标：

1. 将不同来源的课程 PDF 统一转换为稳定的课程文本单元。
2. 保留建图所需的章节结构、页码溯源和必要多模态说明。
3. 降低目录页、版权页、页眉页脚、OCR 噪声对 GraphRAG 建图质量的干扰。
4. 为 `pdf_ingest` 与 `graphrag_pipeline` 之间建立可测试、可演进的字段契约。

## 2. 输出原则

1. 标准输出格式唯一为 `JSON`。
2. 标准输出面向“语义检索与知识建图”，不追求视觉排版还原。
3. 每条记录必须满足：
   - 语义相对完整
   - 标题层级明确
   - 页码可追溯
   - 字段稳定可验证
4. `Markdown` 仅作为调试和人工审阅的派生视图，不作为正式契约。
5. 规范不预设所有资料都是“教材”，应兼容讲义、实验指导、课程说明、参考资料等无编号或弱结构文档。

## 3. 标准输出 Schema

标准文档单元称为 `NormalizedDocument`，字段如下：

```json
{
  "id": "os:计算机操作系统.pdf:ch2-sec2.1",
  "source_file": "计算机操作系统.pdf",
  "document_type": "textbook",
  "course_id": "os",
  "chapter": "第二章 进程的描述与控制",
  "section": "2.1 前趋图和程序执行",
  "subsection": null,
  "heading_level": 2,
  "heading_path": [
    "第二章 进程的描述与控制",
    "2.1 前趋图和程序执行"
  ],
  "content": "规范化后的正文文本",
  "page_start": 3,
  "page_end": 3,
  "metadata": {
    "source_page_indices": [2],
    "has_table": true,
    "has_equation": true,
    "has_image": true,
    "table_count": 1,
    "equation_count": 1,
    "image_count": 1
  }
}
```

字段说明：

1. `id`
   - 标准文档唯一标识。
   - 推荐由 `course_id + source_file + heading_path + chunk_index` 组成。
2. `source_file`
   - 原始 PDF 文件名。
3. `document_type`
   - 枚举值：`textbook | slides | syllabus | lab | notes | exam | reference | unknown`
   - 当前实现优先看文件名中的显式线索；若文件名不足以判断，则再看文档前部正文中的内容信号。
   - 文件名与正文都没有足够信号时，保留为 `unknown`，避免强行误判。
4. `course_id`
   - 课程标识。
5. `chapter`
   - 章级标题，没有则为 `null`。
6. `section`
   - 节级标题，没有则为 `null`。
7. `subsection`
   - 小节级标题，没有则为 `null`。
8. `heading_level`
   - 当前文档单元对应的标题层级，从 `1` 开始。
9. `heading_path`
   - 从高到低的完整标题路径。
10. `content`
    - 供 GraphRAG 建图的正文文本。
11. `page_start` / `page_end`
    - 使用 **1-based PDF 顺序页码**。
    - MinerU 原始 `page_idx` 保留在 `metadata.source_page_indices` 中。
12. `metadata`
    - 辅助信息，仅保留建图辅助和溯源信息，不承载主语义字段。

## 4. 标题与层级规范

1. 标题识别优先级：
   - MinerU 原生标题块
   - 标题编号模式补充
   - Markdown 标题补充
2. 不允许只依赖启发式正则决定全部标题层级。
3. 必须显式区分：
   - `chapter`
   - `section`
   - `subsection`
4. 标题文本标准化要求：
   - 去除目录页码
   - 去除目录点线
   - 去除尾部装饰性符号
   - 保留原始语义编号
5. 目录页中的标题不得作为正式章节边界。
6. 若文档不存在明确章级标题，可从节级开始构造 `heading_path`。

当前阶段 4 已落实的标题标准化规则：

1. 标题尾部的目录页码会被剥离，例如 `第二章 进程的描述与控制 32` 会规范为 `第二章 进程的描述与控制`。
2. 标题中的目录点线会被剥离，例如 `第三章 处理机调度与死锁 ..... 85` 会规范为 `第三章 处理机调度与死锁`。
3. 构造 `heading_path` 时按“同级弹栈、下级入栈”处理，避免缺失章级标题时，后续节标题错误继承上一个节标题。
4. 对非教材类文档，若缺少 `第x章` / `1.2` 这类编号标题，则允许使用 MinerU 的 `TITLE + text_level` 或 markdown 标题层级作为章节边界。
5. 当标题既存在明显编号，又存在 MinerU `text_level` 时，优先以编号模式决定真实层级，避免 MinerU 将多级标题压平成同一级。

## 5. 文本块粒度规范

1. 标准建图单元默认以“节 / 小节级”为主。
2. 同一条记录的 `content` 应保持语义连贯。
3. 不应将整章直接作为一个超长文档，也不应切成单句碎片。
4. 建议目标范围：
   - 优先控制在 `500-2500` 个中文字符
   - 超长内容沿段落或子标题继续拆分
5. 拆分后仍需保留相同的 `heading_path` 和可追溯页码。

当前阶段 5 已落实的粒度规则：

1. 对超长 section 文档启用软切分，默认目标上限约为 `2200` 字符。
2. 若单段过长，则继续按句子或分句边界切分，而不是生硬地整段截断。
3. 软切分只默认作用于 `section` 文档；`page` 文档不会因为默认软切分被拆散。
4. 切分结果在 `metadata` 中记录 `chunk_index`、`chunk_total`、`chunk_char_count` 和 `chunk_strategy`。

## 6. 表格规范

1. 表格主输出统一为“语义化文本”，不以 HTML 或 Markdown 作为正文主格式。
2. 推荐形式：
   - 先输出表题
   - 再按行输出“字段=值”或“对象: 属性=值”
3. 表注保留在表格文本后部。
4. 原始 HTML、图片引用等放入 `metadata`，不直接污染正文。
5. 同类表格在不同课程资料中应采用同一转换规则。

当前阶段 4 的正文呈现约定：

1. 不再在正文中保留 `[TABLE] ref=...` 占位符。
2. 如果 MinerU 已经给出语义化表格文本，则直接复用该文本。
3. 若仅有 HTML 表格，则退化转换为可读文本，再进入正文。
4. 表题作为正文前缀保留，表注以 `表注：...` 形式追加。

## 7. 公式规范

1. 行内公式并入正文，不单独切块。
2. 独立公式保留独立段落，但与上下文组成同一文档单元。
3. 公式主表示优先保留 `LaTeX`。
4. 若无法保留高质量 LaTeX，则退化为可读文本。
5. 不允许正文中大量残留未清洗的控制序列噪声。

当前阶段 4 的正文呈现约定：

1. 独立公式统一以 `公式：...` 前缀进入正文，便于下游区分正文句子与公式段落。
2. 行内公式仍跟随原上下文输出，不额外拆块。

## 8. 图片规范

1. 默认只保留对知识理解有帮助的图片说明。
2. 封面图、装饰图、无说明插图默认丢弃。
3. 图片处理要求：
   - 有图题时保留图题
   - 有图注时保留图注
   - 若图中承载关键概念，可补一句简短说明
4. 图片路径、原始引用等放入 `metadata`。
5. 不应长期依赖 `[IMAGE] ref=...` 占位符作为正文主内容。

当前阶段 4 的正文呈现约定：

1. 不再在正文中默认保留 `[IMAGE] ref=...` 占位符。
2. 仅当图片存在图题或图注时，才向正文输出对应说明文本。
3. 图注统一以 `图片说明：...` 形式追加；原始图片引用继续保留在 `metadata`。

## 9. 噪声清理规范

必须清理：

1. 页眉、页脚、页码
2. 目录页及其伪标题
3. 版权页、CIP、ISBN、印次、出版社信息
4. 连续重复块
5. 跨页重复短文本
6. 装饰线、纯数字噪声、无意义符号块
7. OCR 误识别导致的明显乱码

当前阶段 4 已落实的目录页过滤规则：

1. 若页面中出现 `目录`、`contents` 等目录标记，且同时存在多条“标题 + 点线/空白 + 页码”模式，则整页按目录页处理。
2. 被判定为目录页的页面，在章节聚合前整体移除，避免切出伪章节。

当前阶段 5 已落实的前置噪声页过滤规则：

1. 仅对文档最前面的少量页面做“出版/版权噪声页”检测，避免误伤正文。
2. 只有命中多个强信号，如 `CIP`、`ISBN`、`版权声明`、`出版社`、`版次/印次` 等，才会整页过滤。
3. `前言`、`摘要`、`课程说明`、`学习目标`、`实验目的` 等导读型页面默认保守保留。

## 10. 预处理流程

目标流程如下：

1. MinerU 原始结果归一化为内部 `Block`
2. 标题识别与层级校正
3. 目录、版权、页眉页脚和重复噪声清理
4. 表格、公式、图片统一语义化处理
5. 构建 `chapter/section/subsection/heading_path`
6. 按目标粒度切分为 `NormalizedDocument`
7. 从标准文档投影生成 GraphRAG 兼容 JSON
8. 输出统计报告与异常样本清单

## 11. GraphRAG 兼容策略

为兼容当前 GraphRAG 输入约束，允许从标准文档投影生成轻量记录，但标准文档是唯一上游事实来源。

GraphRAG 投影至少需要：

1. `title`
2. `text`
3. `course_id`
4. `source_file`
5. `page_start`
6. `page_end`
7. `section_level`

后续若 GraphRAG 侧允许读取更多顶层字段，再逐步提升 `document_type`、`chapter`、`section` 等字段的透传比例。

## 12. 基线样例与回归样本

阶段 1 + 阶段 2 的基线样例存放于：

1. `tests/fixtures/normalization/content_list_with_toc.json`
   - 用于复现目录页伪章节、标题尾页码、多模态混合块等问题
2. `tests/fixtures/normalization/normalized_doc_v1.json`
   - 用于冻结目标输出字段和页码定义

后续新增课程样例时，至少覆盖：

1. 教材类 PDF
2. PPT / slides
3. 实验指导
4. 多表格文档
5. 多公式文档
6. 图片较多文档

## 13. 验收标准

阶段性完成标准如下：

1. 所有课程资料输出统一 Schema
2. 标题不再包含目录页码和点线
3. 目录页不再切出伪章节
4. `content` 中显著减少版权页、页码、目录和 OCR 噪声
5. 表格、公式、图片处理在不同资料中保持一致
6. 每条记录都能通过字段校验与页码追溯
7. 存在自动化测试覆盖字段完整性、标题切分、目录过滤和投影兼容性

## 14. 抽样验收工具

当前提供导出验收脚本：

```bash
python scripts/pdf_processor/export_audit.py <文件或目录> --output /tmp/export_audit_report.json
```

用途：

1. 对 `normalized_docs.json` 或 `section_docs.json` 做全量扫描。
2. 汇总占位符残留、页码异常、标题尾噪声、超长/超短文档等问题数量。
3. 为每类问题保留少量抽样样本，便于人工快速复核。

推荐在以下场景使用：

1. 对单门课程导出结果做验收。
2. 对课程目录做批量扫描，形成阶段性质量报告。
3. 在修改清洗或切分规则后，对比新旧问题分布是否改善。
